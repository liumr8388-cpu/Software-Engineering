# 花卉交易管理系统

这是一个用于软件工程课程设计演示的花卉交易管理系统，包含前端页面、本地后端服务和 SQLite 数据库。

## 推荐运行方式

双击运行：

```text
run.bat
```

系统会自动打开：

```text
http://127.0.0.1:8000
```

也可以在终端中手动运行：

```bash
python server.py
```

## 数据库说明

启动 `server.py` 后，系统会在项目根目录自动生成 SQLite 数据库文件：

```text
flowers.db
```

数据库表结构见：

```text
database.sql
```

页面右上角会显示当前存储模式：通过 `http://127.0.0.1:8000` 访问时显示“SQLite 数据库”；直接双击 `index.html` 时显示“浏览器本地”。

## 功能模块

- 花卉信息管理：新增、编辑、删除花卉商品，维护 SKU、价格、供应商、库存阈值等信息。
- 客户下单流程：客户浏览花卉、加入购物车、提交订单，系统生成订单并通知管理员。
- 订单管理：支持订单创建、付款、发货、交付、取消等生命周期状态流转。
- 库存管理：支持库存补货、报废、销售扣减记录，并在库存低于阈值时自动生成补货预警。
- 客户管理：维护客户联系方式、购买历史、偏好和会员等级。
- 销售统计：按月度、季度、年度统计销售额、订单量和畅销花卉。
- 供应商管理：维护供应商联系方式、合作协议和付款记录。

```text

├── app.js          # 前端交互逻辑

├── seed-data.json  # 初始演示数据
├── server.py       # 本地 Web 服务和数据库接口

```

## 课设说明

本项目覆盖需求规格说明中的核心业务：商品库、订单全生命周期、库存阈值预警、客户档案、销售统计和供应商协同。后端使用 Python 标准库实现 HTTP 服务，数据库使用 SQLite，便于在课程设计环境中直接运行和展示。
